Редактирование ресурсов

При помощи редактирования ресурса можно изменять состояние системы.

Полное редактирование

Полное редактирование производится при помощи запроса по пути PUT /api/{v}/{resource}/{id} контроллера ресурса.

В теле такого запроса передается полная модель ресурса, которая полностью заменит текущее состояние ресурса по указанному id.

Warning

У модели ресурса чаще всего есть "тяжелые" вложенные свойства, которые могут представлять собой другой ресурс. Также это может быть коллекция некоторых ресурсов.
Такие свойства, или короче говоря, любые свойства непримитивного типа (т.е. не string, int, boolean и т.д.) - могут быть исключены из модели, передаваемой в теле запроса, таким образом исключенные свойства останутся неизменными после выполнения запроса.
Рекомендуется исключать (присваивать этим свойствам значение null) такие свойства из тела запроса, если они не были изменены. Это поможет значительно ускорить выполнение запроса.

Warning

🆕 При изменении свойств коллекций путем полного редактирования(и частичного, но изменения проивзодятся самим сервером путем применения переданных правил JSONPATCH) - сервер приведет её к такому же состоянию, которое будет передано в теле. Т.е. сущности которые не были переданы - удалятся, а новые(у которых id=0) - добавятся.

Частичное редактирование

Частичное редактирование производится при помощи запроса по пути PATCH /api/{v}/{resource}/{id} контроллера ресурса.

В теле такого запроса передается массив модели JSONPATCH, которая декларативным способом, в виде действий, описывает изменения модели.

Warning

Важно, при изменении свойств коллекций (массив в JSON) не изменять порядок элементов этой коллекции, т.к. JSONPATCH иденцифицирует элементы коллекций по индексу, и если элементы будут перемешаны относительно полученного порядка с сервера - запрос отредактирует не тот элемент.
Если в UI требуется выводить элементы в сортированном виде, это нужно делать при помощи методов которые не изменяют состояние свойства.

Warning

Если для запроса данных были использованы доп. параметры, которые расширяют модель, то те же самые свойства должны использоваться и в запросе PATCH на частичное редактирование, если были отредактированы затронутые данные (например при редактировании разрешений модели Role, когда разрешения загружаются при доп. параметре includeClaims=true)
