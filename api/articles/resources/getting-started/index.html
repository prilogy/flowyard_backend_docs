Ресурсы и модули / Введение

Основным понятием любого RESTful API является ресурс.

Ресурс - это объект с типом, связанными данными, отношениями с другими ресурсами и набором методов, которые с ним работают. Он похож на экземпляр объекта в объектно-ориентированном языке программирования с тем важным отличием, что для ресурса определены чаще всего только несколько стандартных методов ( соответствующих стандартным методам HTTP GET, POST, PUT, PATCH и DELETE), а экземпляр объекта обычно имеет много методов.

Note

В данном разделе будет приведена информация об особенностях конкретных ресурсов и модулей. Спецификация моделей находится в Swagger.

🆕 Базовые и часто встречаемые свойства

  • id - Id
  • dateCreated - дата создания ресурса
  • dateUpdated - дата обновления ресурса
  • createdByUserId - Id пользователя, которым был создан ресурс

Некоторые свойства могут быть скрыты в DTO, но присутсвуют в настоящей модели, поэтому по надобности их можно добавить в DTO.
