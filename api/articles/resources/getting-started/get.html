Получение списка ресурса

Запрос по пути GET/ api/{v}/{resource} имеет несколько особенностей

Пагинация

Имеются следующие параметры:

Название Значение по умолчанию Домен Описание
page 1 >0 Номер страницы
pageSize 10 >0 Размер страницы
🆕 orderBy date_created date_created / name * Правило сортировки выдачи.
date_created - по дате создания. name - по названию (если такое свойство есть в модели ресурса или задано сервером дополнительно)
🆕 order desc desc / asc Порядок выдачи.
Для date_created asc возвращает объекты с id 1, 2, 3 (по порядку)
desc возвращает объекты с id 3, 2, 1 (в обратном порядке) и т.д. для остальных правил orderBy

* - возможны дополнительные значения в зависимости от конкретного ресурса. Если такие значения есть, об этом указано на страницах ресурсов.

Такие параметры передаются в query string URI запроса.

Фильтрация

В дополнение к параметрам пагинации могут идти параметры фильтра ресурса. У каждого ресурса может быть свой, уникальный набор параметров фильтра.

Параметры фильтров также задаются через query string. Подробная спецификация фильтров находится в Swagger.

Дополнительные параметры

В дополнение к ранее описанным параметрам могут идти дополнительные параметры. У каждого ресурса может быть свой, уникальный набор дополнительных параметров.

Обычно через доп. параметры можно дополнить получаемую модель некоторыми данными.

Дополнительные параметры также задаются через query string. Подробная спецификация фильтров находится в Swagger.
