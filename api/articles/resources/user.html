Пользователь (User)

Одна из основных сущностей системы. Определяет данные пользователя, его компании и роли.

Свойство Тип Домен Описание
name string Геттер, который формирует ФИО пользователя в соответствии с текущей культурой
color string #xxxxxx Цвет пользователя. В него красится аватар если не задана картинка аватара.
isEmailConfirmed bool Указывает, подтвержден ли Email
types UserType[] Типы пользователя
currentCultureKey string Ключ текущей культуры пользователя
companies User2Company[] Содержит компании, в которых состоит пользователь
currentCompanyId int Id компании текущего контекста
timeZoneFromUtc int -12...12 Таймзона
avatar UserAvatar Аватар пользователя. Id статического файла с типом image указывается в userAvatar.imageId
🆕 contact UserContact Контактные данные пользователя. contact.name - название (getter), contact.key - ключ, contact.value - значение.
Ключи по запросу GET /user-contact-type контроллера Helper

🆕 Удаление (выход) пользователя из компании

Происходит при помощи роута DELETE {userId}/{companyId} контроллера User.

  • Удалять других пользователей (не себя) из компании можно только в рамках текущей компании и текущего контекста. Т.е. нельзя удалить пользователя из компании #1 пока находишься в компании #2
  • "Выходить" от лица текущего пользователя можно из любой компании, к которой прикреплен пользователь
Note

После выхода из компании от лица текущего пользователя нужно делать повторную аутентификацию с refresh token.

UserShort

Упрощенная модель User для более быстрых и легких запросов. Со спецификацией можно ознакомиться в Swagger.

Note

Для этой модели есть отдельный контроллер UserShort.
