Комментарий к проблеме (IssueComment)

Для добавления нового комментария нужно создавать новую сущность IssueHistory, где в issueHistory.comment прикрепляется сам комментарий.

Свойства

Свойство Тип Домен Req. Описание
text string ✅ Текст комментария
attachments IssueCommentAttachment[]? Вложения (файлы) к комментарию. Подробнее о статических файлах
🆕 recipients IssueCommentRecipient[]? Список адресатов
🆕 isMessage bool (getter) Является ли комментарий сообщением

Вложение комментария (IssueCommentAttachment)

Свойства

Свойство Тип Домен Req. Описание
fileId int >0 ✅ Id статического файла
order int >=0 ✅ Порядок вложения при загрузке пользователем

🆕 Адресат (IssueCommentRecipient)

Если issueComment.recipients не пустое(внутри хотя бы один элемент), то комментарий является сообщением (тем не менее для определения этого использовать issueComment.isMessage)

Свойства

Свойство Тип Домен Req. Описание
userId int >0 ✅ Id пользователя - адресата

Добавление сообщения

При отправке сообщения происходят следующие действия над issueComment:

В issueComment.recipients кладётся объект с userId равным:

  • Если исполнитель задачи пишет соообщение - то issue.createdByUserId
  • Если отправитель задачи пишет сообщение - то issue.executorUserId
