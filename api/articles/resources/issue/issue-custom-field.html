Кастомное поле проблемы Issue (IssueCustomField)

Сущность нужна для установки кастомных полей Issue разных типов.

Note

Получить список кастомных полей с их типами можно через роут GET/ issue/custom-field контроллера Helper.

Свойства

Свойство Тип Домен Req. Описание
🆕 name string? Название кастомного поля. Прикреплятся по ключу при возврате с сервера
issueId int >0 Id Issue (прикрепляется автоматически если вложен в Issue)
key string ✅ Ключ кастомного свойства
valueType string Типы полей ✅ Тип поля

А также свойства ниже:

Типы полей и соотвествие свойствам модели

Ключ указывается в свойстве issueCustomField.valueType.

Ключ Тип Свойство в IssueCustomField
bool bool valueBool
string string valueString
int int valueInt
dateTime DateTime valueDateTime
Important

При несоответсвии типа и указанного свойства сервер вернет ошибку.

Note

Сейчас используются только поля типа bool.

Флоу

  1. Получаем все виды кастомных свойств по роуту GET/ issue/custom-field контроллера Helper.
    Сохраняем их в глобальный стейт, и в следующий раз из него же берем, без повторного запроса
  2. Сопоставляем объекты коллекции issue.fields с полученными данными в пункте 1. по ключу (key) и отображаем интерфейс с полученным текстом и контролом, сопоставленным по типу поля
