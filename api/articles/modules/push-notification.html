Мобильные оповещения

Конфигурация

При аутентификации

После успешной аутентификации с паролем - отправить запрос по пути POST /fcm-token контроллера Authorized, где в модели указать текущий FCM токен и RefreshTokenId

При выходе

При выходе из аккаунта - отправить запрос по пути DELETE /fcm-token/{token} контроллера Authorized, где в роуте вместо параметра {token} положить текущий FCM токен.

Типы оповещений

Тип оповещения будет храниться под ключем notification_type

Для Issue

Как идентифицировать - notification_type содержит слово "issue"

Ключи - значения

Ключ Тип Описание
issue_id int Id Issue. При нажатии на оповещение - открывать Issue с этим Id
company_id int Id компании. Если он не равен id текущей компании - открывать диалог с текстом * и предлагать переход в другую компанию для просмотра Issue

* - заголовок - "Переключить компанию?", тело - "Чтобы посмотреть эту задачу, нужно переключить компанию", действия да/нет
