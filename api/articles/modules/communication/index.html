Получение доски

В свойстве initialInfo.boards модели InitialInfo (получаемой после аутентификации) хранится список доступных на текущий момеент досок. Модель не содержит список статусов, поэтому для более полной информации нужно отдельно запрашивать доску по Id через контроллер Board.

Warning

На данный момент существует только по одной доске на компанию.

Получение Issue

Для получения задач для доски нужно использовать запрос GET контроллера Issue с использованием фильтра. Значение boardStatusId фильтра позволяет получить задачи по конкретной колонке(статусу) доски.

При первичной инициализации задачи из колонок запрашиваются по-отдельности. Т.е. на каждый статус делается по запросу GET контроллера issue с фильтром boardStatusId.

Note

Рекомендованное значение пагинации <=20

Note

Для задач которые отображаются на доске рекомендуется запрашивать минимальную модель для улучшения производительности.

Warning

Возможно добавление роута для запроса облегченной модели IssueShort для улучшения производительности.

Note

Для определения размера виртуального скроллера можно использовать свойство модели пагинации pagingModel.totalItems

Смена статуса

Новые статусы присваиваются при помощи создания новых IssueHistory через соответствующий контроллер. Сам id статуса указывается в свойстве issueHistory.status.boardStatusId.

Доступные для переключения статусы можно получить по роуту GET /{id}/calculated контроллера Issue, либо присоединить их к модели Issue в обычных запросах ресурсов при помощи доп. параметров.

Warning

Не рекомендуется присоединять действия и следующие статусы к модели при запросах пагинации. Может произойти значимая потеря производительности.
