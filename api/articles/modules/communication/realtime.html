1. Подключение к хабу SignalR

Подключение к хабу происходит по пути api/v1/hub/issue при помощи фронтенд SignalR клиента.

При подключении передавать query string параметр access_token равный JWT токену (без Bearer префикса)

Note

Хаб - своеобразное простанство имен для SignalR.

2. Установка Id текущей доски

После подключения к SignalR хабу отправить сообщение с именем SetCurrentBoardId с одним агрументом типа int - id текущей доски

Warning

Также слать это сообщение каждый раз при смене доски (на текущий момент смены доски нет).

3. Обработка сообщения

После успешных проделанных действий клиент будет получать уведомления об изменениях на доске.

Сообщение имеет имя IssueUpdate.

Сообщение передает модель типа IssueUpdateMessage.

Свойства IssueUpdateMessage

Свойство Тип Описание Обработка
boardId int Id доски для которой актуально обновление -
updatedBoardStatusIds int[] Id BoardStatus(колонок доски) Если колонка(-и) с id из свойства в поле зрения - перезапустить пагинацию для этих колонок. Иначе сбросить пагинацию чтобы при следующем входе она запросилась заново

type string Тип обновления Типы обновлений

  • issue.added - Issue добавлена
  • issue.updated - Issue обновлена
  • issue.deleted - Issue удалена. Доп. обработка - если открыт дровер с задачей по IssueId - закрыть дровер.
  • 🆕 issue.history.updated - история Issue обновлена
