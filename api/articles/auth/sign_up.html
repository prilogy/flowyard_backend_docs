Регистрация через приглашение (CompanyInvite)

Приглашения бывают одноразовыми(companyInvite.isReusable == false), и иначе - многоразовыми. При наличии кода от приглашения(companyInvite.code), регистрация проходит с помощью следующих роутов

Id URI Описание
1 GET/ api/v1/authentication/sign-up/company-invite/{code} Проверка кода на валидность, возвращает модель компании, в которую будет добавлен пользователь после регистрации
2 POST/ api/v1/authentication/sign-up/company-invite Регистрация
Note

Код приглашения не чувствителен к регистру

Tip

Проверка валидности кода (и, например, вывод информации о компании из приглашения) - процедура опциональная, и направлена по большей части на улучшение UX, когда сначала у пользователя появляется экран с одним текстовым полем, и кнопкой "Продолжить", после чего, в случае валидности кода - переход на экран с информацией о компании и двумя полями ввода (почты и пароля).

Сценарии использования

🆕 Ссылка приглашения

Для любого из вариантов в companyInvite.url генерируется ссылка на регистрацию, где query параметры:

  • code - обязательный параметр, хранит код приглашения
  • email - опциональный параметр, хранит email, указанный при создании приглашения

Неаутентифицированное состояние

Создается новый пользователь с указанными данными, на фронте после регистрации автоматически происходит вход.

Warning

При переходе на регистрацию по ссылке приглашения - автоматически вводить код из ссылки и email, если он указан.

Аутентифицированное состояние

Текущий пользователь прикрепляется к компании из приглашения.

🆕 Флоу такой же как и при регистрации из неавторизованного состояния, за исключением того, что из модели запроса будет браться только код, остальные значения будут проигнорированы.

Warning

Требуется повторная аутентификация с refresh token.

Warning

🆕 При переходе на регистрацию по ссылке приглашения - открывать диалог с добавлением новой компании, где автоматически вставлять код из ссылки.
