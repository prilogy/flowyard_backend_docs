Контекст локализации

При помощи хедера Culture задается локализация приложения.

Этому хедеру также соответствует свойство user.currentCultureKey, которое прикрепляется в результе аутентификации. Важно чтобы и хедер, и это свойство были синхронизированы.

Note

🆕 Указывается при любых видах аутентификации.

Warning

Если не прикрепить хедер Culture при действиях аутентификации, может произойти неожидананная смена локализации, поэтому рекомендуется всегда прикреплять хедер Culture при аутентификации и синхронизировать значение приходящее с сервера.

Note

Доступные локализации можно получить при помощи запроса GET /api/{v}/helper/culture.

Действия при смене локализации

Инициатором смены локализации является клиент.

Для смены текущего контекста компании нужно:

  1. Установить хедер Culture в значение ключа языка, на который происходит переключение.
  2. Сделать повторную аутентификацию с помощью refresh-token, который возвращает данные текущего контекста, где в user.currentCultureKey будет ожидаемым значение которое было установлено в хедер.
  3. В случае успеха нужно сбросить/обновить глобальное состояние приложения данными, полученными из запроса в шаге 2.
Warning

Если указан несуществующий ключ языка - сервер вернет ошибку, таким образом, ранее установленный хедер нужно откатить на предыдущее значение.
